# adal-youtube-script-writer

Why the first 15 seconds matter

The steepest viewer drop-off on YouTube happens right around the 15-second mark. Whoever is still there at 0:15 mostly stays. Whoever isn't, leaves.
The skill is tuned around a real gap: scripts that land a concrete value claim before 0:15 hold noticeably more viewers than ones that don't (roughly 52% vs 44% in the reference data the skill is built on). So it forces a real value claim into the first 15 seconds of every script.
This is also why the hook opens on a real frustration instead of a greeting or a logo bumper. A "hey guys, welcome back" burns the exact seconds that decide whether anyone keeps watching.

Why the skill is built the way it is

Teaching leads, retention serves it. Every video has a concept beat (what it is, why it matters) and a show beat (here it is working). The four video types just change the ratio. Basics leans concept, demos lean show, model releases sit in between.
It keeps the retention goals (hook fast, value by 0:15, a mid-video re-hook, an end CTA pointing to the next video) but drops the rigid formula that used to come with them. Ranking your points by value and chopping sentences into short fragments is what made earlier drafts read as robotic. Structural symmetry and staccato fragments are the real fingerprint of AI writing, more than any single word.
Voice is direct and human, not casual. Forced casual tone reads as try-hard from anyone who doesn't already write that way, so the skill aims for plain and clear and lets the reasoning carry the teaching.
Structure and packaging follow Cursor's AI Foundations series: name the friction, build the mental model, give one analogy, then show it on something real.

How to use:

Trigger it by naming your topic and, ideally, the video type (Basics, feature, model release, or demo).
It works in three stages. It asks up to three short questions, writes a rough spoken-word draft, then writes the full formatted script after you approve.
The three questions are a thinking tool, not a form. If your first message already answers them with real specifics, it skips straight to the draft. Vague input gets questions, specific input gets a draft.
It will not make up benchmark numbers, model names, or feature claims. If a stat isn't confirmed locked, it leaves it out and flags it. Feed it the real numbers.
Read every output out loud before filming. Make sure it sounds like YOU -- human, educational but still conversational and not like a robot.
Think of a real use case that would be compelling for a viewer.
